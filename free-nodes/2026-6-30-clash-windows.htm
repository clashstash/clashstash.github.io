一、Shadowrocket到底是什么？

1.1 定义与定位

Shadowrocket是一款运行在iOS系统上的网络代理工具，本质上是一个“客户端软件”。它本身并不提供网络代理服务，而是作为一个“桥梁”，将用户与各种代理服务器连接起来。用户可以把它理解为一把“钥匙”，而代理服务器则是“门”，只有钥匙和门配合，才能真正实现访问受限内容的目的。

1.2 核心功能

Shadowrocket支持多种主流的代理协议，包括但不限于：

  • Shadowsocks：最经典的加密代理协议，轻量且高效
  • Vmess：V2Ray的核心协议，具备更强的抗检测能力
  • Trojan：伪装成HTTPS流量的新型协议，隐蔽性极高
  • HTTP/HTTPS代理：基础的代理协议，兼容性广泛

这种多协议支持的特性，使得Shadowrocket能够适应不同的网络环境和需求，这也是它受到追捧的重要原因。

1.3 与VPN的本质区别

很多人习惯将Shadowrocket称为“VPN”，但这种说法并不准确。传统VPN会在设备层面创建虚拟网络接口，所有流量都经过加密隧道传输；而Shadowrocket工作在应用层，通过配置代理规则，只对特定流量进行转发。这种设计使得Shadowrocket更加灵活，资源占用更小，但也意味着它需要用户具备一定的配置能力。

  1. 用户发起请求：你在浏览器中输入一个网址，比如 google.com

FAQ

Hiddify Next 如何设置不同用户的带宽限制？
管理员可为每个用户设置独立的带宽上限和流量配额。超过配额后可自动阻断或降速，确保服务器资源合理分配，避免单个用户占用过多带宽而影响其他用户体验。
Matsuri 的多节点测速机制是怎样的？
Matsuri 使用并发测速机制，可同时对多个节点进行延迟与带宽测试。测试结果会自动排序，并可根据延迟或速度自动切换最优节点，提升连接体验。
SSR 的端口复用有什么风险？
SSR 支持多端口配置以复用服务，但若端口过多或设置冲突，可能被扫描工具识别或误判为异常服务。建议保持配置简洁，并合理分配端口范围以减少暴露风险。
如何检查 Clash 连接失败的日志以定位问题？
打开 Clash 的日志面板或查看配置目录下的 log 文件，观察报错信息如 DNS 解析失败、节点拒绝连接或 TLS 握手错误。根据关键字（例如 timeout、refused、certificate）逐项排查并比对服务器配置。
V2Ray 的路由规则可设置哪些匹配条件？
路由规则支持按域名、IP 地址、端口、协议类型或用户标签进行匹配。结合策略组，可实现国内直连、国外代理或特定节点定向分流，满足不同网络需求。
Clash 的 Rule Provider 功能有哪些好处？
Rule Provider 可通过远程订阅自动更新规则和节点，减少手动维护。结合策略组，可实现分流规则和节点的自动同步，保证分流效率和网络可用性。
SSR 的 obfs 混淆原理是什么？
SSR 的 obfs 模块通过模拟常见流量特征（如 HTTPS 或 HTTP 请求头）来掩盖加密流量。这样可让代理流量看起来像正常的网页访问，从而有效降低被检测和封锁的风险。
QuantumultX 的广告屏蔽原理是什么？
QuantumultX 使用分流规则与脚本结合的方式实现广告屏蔽。通过重写 HTTP 请求和阻止特定域名，它能有效去除 App 内广告或网页弹窗，提升浏览体验。
Shadowrocket 支持哪几种代理协议？
Shadowrocket 支持多种协议，包括 Shadowsocks、Vmess、VLess、Trojan、HTTP、SOCKS5 等。它还可解析部分 QuantumultX 与 Clash 的订阅格式，适合 iOS 用户进行全协议代理管理与切换。
Sing-box 的性能比 Clash 好吗？
Sing-box 在性能优化上更为出色，特别是在高并发连接下延迟更低。它采用 Go 语言编写并优化了内存占用，比传统 Clash 内核运行更轻盈，尤其适合服务器端部署或移动端设备使用。
